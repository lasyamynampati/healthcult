# Incident Response Runbook

## Trigger conditions
- Chat safety policy violation.
- OCR pipeline repeated failures.
- Prediction endpoint abnormal error spikes.
- Suspected privacy or unauthorized data access incident.

## Response steps
1. Triage and classify severity.
2. Contain impact (disable affected feature flags or routes).
3. Preserve logs and audit evidence.
4. Notify stakeholders and compliance owners.
5. Patch, validate, and publish post-incident report.
